# 面试助手前端视觉优化设计

## 目标

在不改变音频采集、字幕、生成建议和追问接口的前提下，将当前偏原型的界面升级为更舒适、清晰的桌面端 AI 助手界面。

用户确认采用“清爽智能”视觉方向：冷白底色、靛蓝主色、淡紫灰分层；所有顶部操作按钮常驻；内容区固定为左右双栏。

## 范围

### 包含

- 顶部工具栏重构：品牌标识、采集状态、开始或停止采集、生成建议、清空上下文全部常驻。
- 左侧“实时字幕”卡片：标题栏、自动滚动说明、空状态、定稿字幕和识别中字幕具有明确层级。
- 右侧“回答建议”卡片：建议内容、加载状态、错误状态、追问记录和追问输入区按阅读优先级重排。
- 全局视觉规范：背景、卡片、按钮、状态色、排版、阴影、滚动条和错误提示统一。
- 浏览器窗口变窄时保持字幕在左、回答建议在右；允许页面横向滚动，不变为上下堆叠。

### 不包含

- 不改变 API、WebSocket、音频采集、字幕状态或聊天状态逻辑。
- 不增加设备选择、账户、主题切换或移动端单列布局。

## 布局与交互

页面由高度固定的顶部工具栏和主内容区组成。主内容区使用固定左右双栏：左栏约占 44%，放置实时字幕；右栏约占 56%，放置回答建议及追问。

顶部工具栏的采集按钮为主操作：未采集时显示“开始采集”，采集时显示“停止采集”。生成建议保持靛蓝强调样式；清空为低强调的次要按钮。采集中的状态胶囊显示蓝色圆点与“正在采集”，没有采集时显示中性状态。

左侧字幕卡片在无内容时显示引导文案；定稿字幕按段落呈现；实时识别内容使用淡靛蓝背景，避免与定稿内容混淆。右侧建议卡片将建议正文置于淡色建议容器中，追问历史使用角色气泡；输入框固定在卡片底部。

## 视觉规范

- 背景：`#F7F8FD`，配合极轻的靛蓝渐变光晕。
- 主色：`#566ADD`；信息与进行中状态使用相近蓝色系。
- 卡片：白色或半透明白色背景，`#E5E9F7` 描边，14–20px 圆角，柔和阴影。
- 文本：主文本深蓝灰，辅助文本蓝灰；正文行高不低于 1.7，保证长时间阅读舒适。
- 按钮：主按钮实心靛蓝，其他按钮为白底描边；提供悬浮、禁用和聚焦状态。

## 组件边界

- `App.tsx`：只负责页面壳、错误提示和现有业务钩子的编排。
- `Controls.tsx`：渲染顶部工具栏及采集状态。
- `SubtitlePanel.tsx`：渲染字幕卡片与自动滚动内容。
- `SuggestPanel.tsx`：渲染建议卡片、追问历史和输入区。
- `index.css`：集中放置设计令牌、全局重置、布局与组件样式。

业务组件仍通过既有 props 通信，不引入新的全局状态。

## 错误处理与可访问性

- 采集、字幕或聊天错误统一置于顶部工具栏下方的醒目错误条，避免挤占双栏内容。
- 可点击元素保持原生按钮和输入框语义；键盘焦点有可见轮廓。
- 不以颜色作为唯一状态表达：采集状态含文字，识别中内容含明确说明。

## 验证

- 执行 `npm run build`，确认 TypeScript 编译和生产构建通过。
- 在本地页面检查：空状态、开始与停止采集、错误提示、长字幕滚动、生成建议、追问历史与输入区。
- 在桌面和窄窗口检查内容区始终为左右双栏，不出现上下堆叠。
